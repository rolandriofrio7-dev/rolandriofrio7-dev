## Why I build

I spent ~300 hours in a behavioral health and transitional care clinic. I watched people fall through gaps that weren't anyone's fault — care coordination breaking down after discharge, the people holding it together with the fewest tools. That's not a data problem. It's a systemic one, and it's exactly the kind of thing software, built carefully, can help with — augmenting human care, never replacing the judgment at the center of it.

That's what I'm working toward: health, AI, and engineering pointed at the people the system currently misses.

## 🩺 Flagship — IO

**[IO — Caregiver Coordination](https://github.com/rolandriofrio7-dev/care-command-center)**
A non-diagnostic caregiver coordination prototype built from real clinical experience. It tracks whether care-plan tasks happened on time, surfaces drift, and keeps a human in charge of every judgment.
- Python / FastAPI backend, deterministic state machine, full action-level provenance
- Next.js / React / TypeScript frontend
- 546 automated tests (backend, component, end-to-end)
- Synthetic-only, non-diagnostic by design — built in small, verified steps with explicit approval gates
